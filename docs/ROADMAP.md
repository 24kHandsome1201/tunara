# Tunara 产品路线图

## 产品判断

Tunara 已经有终端、会话、Git 审查、远程 SSH、文件浏览和 Agent 状态识别。后续最值得发展的方向是 Agent 工作台，让用户在多个本地和远程任务之间更轻松地知道三件事：当前主线在哪里，风险在哪里，下一步做什么。

本轮功能优先处理低风险高收益的体验：会话置顶、会话概览、会话笔记、工作区洞察、专注模式入口。它们都围绕“少切换、少记忆、少迷路”展开。

## 已落地能力

### 1. 会话置顶

关键会话可以固定在切换列表前面，侧边栏和命令面板都会优先显示。置顶状态进入工作区快照，重启后继续保留。

适合场景：长期 Agent 会话、远程修复会话、构建任务、主线开发目录。

### 2. 会话概览

审查栏新增概览页，展示当前会话状态、Agent、目录、远程信息、最近命令、Git 改动、笔记待办进度和快捷动作。

概览页让用户不用在终端、侧栏、Git 面板之间反复切换，就能判断当前会话是不是值得继续盯着。

### 3. 会话笔记

每个会话都可以写轻量笔记，支持自动保存、待办统计、复制、清空，以及四种模板：今日记录、提交检查、Bug 追踪、橡皮鸭解释。

笔记适合保存排查思路、复盘结论、临时待办和给 Agent 的上下文。

### 4. 工作区洞察

标题栏新增工作区洞察入口，命令面板也可以打开。洞察面板会统计会话数、远程会话、忙碌 Agent、未读会话、Git 改动、需要清理的完成会话，并给出当前工作区强度分。

它还会生成一个每日代号，比如 Copper Otter、Solar Tuna，让长时间开发多一点趣味。

### 5. 专注模式入口

命令面板和工作区洞察面板加入专注入口，可以一键收起侧栏和审查栏。命令面板里再次触发可以恢复两侧面板。

### 6. 协作简报

工作区洞察可以复制一段纯文本简报，包含会话数量、运行状态、Agent 状态、改动摘要和未读数量。这个简报可以直接贴到 issue、PR、聊天窗口或交接文档里。

### 7. 起步工作流

设置里的工作流页新增一键加入模板，覆盖 Git 快照、跑测试、TODO 扫描、大文件巡检和端口排查。用户可以马上得到一组常用命令，再按自己的项目慢慢改。

## 下一阶段建议

### 1. 远程 Git 审查

让远程 SSH 会话也支持 diff、状态统计和文件搜索。这个方向收益很高，因为 Tunara 已经有远程文件浏览能力，下一步可以把远程修复的闭环补齐。

### 2. Agent Runbook

把常用 Agent 操作沉淀成可执行模板，比如启动、检查、修复、回滚、生成报告、跑测试。Runbook 可以接入命令面板和会话笔记，让工作流可以复用。

### 3. 任务时间线

把命令开始、命令结束、Agent 忙闲、失败退出、Git 改动、通知和笔记串成时间线。排查问题和复盘工作会更省力。

### 4. 一键交接包

把当前工作区状态导出成 Markdown 报告，自动包含最近命令、失败日志、diff 摘要、待办列表和远程信息。适合多人协作和长任务交接。

### 5. 项目健康分

根据测试失败、失败命令、改动规模、长时间未清理会话、未读状态和远程状态给出轻量健康提示。这个功能可以放在工作区洞察里。

### 6. 趣味增强

可以继续加入“橡皮鸭复盘”“每日小任务”“会话称号”“连续清理 streak”等轻量玩法。重点是帮助专注和复盘，避免变成干扰。

## 性能方向

1. 会话列表继续保持 memo 化，过滤、分组、排序要和渲染拆开。
2. Git 状态刷新做队列和去重，避免多会话同时触发重复 IPC。
3. 大 diff 使用虚拟滚动和按文件懒加载。
4. 远程文件搜索需要分页、取消请求和结果缓存。
5. 笔记、布局和工作区快照继续走防抖保存，避免输入时频繁写磁盘。
6. 批量 Git 刷新继续跳过远程会话，避免把 SSH 伪路径交给本地 Git。
