# Conduit 实施文档 — 从零到完整功能

> 版本：2026-06-18　|　配套阅读：《调研-Conduit终端方案.md》《调研-三大难点深入.md》
>
> **目标**：照本文从空目录搭到完整功能。涵盖环境、脚手架、目录结构、每个 Rust 命令、统一 Agent Harness（含完整代码）、Git 集成、前端接线、颜值（macOS 毛玻璃）、分阶段里程碑与验收清单。
>
> **已固化决策**（详见三大难点文档·决策记录）：
> - **D1 "应用补丁"走简单风**：agent 直接写盘，UI 只展示结果 + diff（+ 写盘护栏 + 「丢弃改动」）。
> - **D2 只接 CC + CX 两家**（砍掉 Cursor/CU）；二者均支持 resume。
> - **D3 统一调 CLI**（claude / codex），不引入 SDK。
> - **D4 commit + push 全走系统 `git` CLI**（禁交互+超时+stderr 回显）；status/diff/ahead-behind 用 `git2`。
> - **D5 终态（status/result/error/updatedAt）MVP 必持久化**；逐字 transcript 重放二期。
> - **D6 并发软上限可配**（默认 ≤4，超限**拒绝+提示**，排队二期）。

## 0. 技术栈总览

| 层 | 选型 |
|---|---|
| 框架 | **Tauri v2**（Rust 后端 + Webview 前端） |
| 前端 | **React 19 + TypeScript + Vite**；样式用 CSS 变量（按设计稿 tokens） |

### 3.2 终端模块（复用 terax，命令一览）

直接搬 `modules/pty`（`mod.rs` + `session.rs` + `shell_init.rs` + `scripts/`）。命令契约（前端用）：

| 命令 | 签名 | 说明 |
|---|---|---|
| `pty_open` | `(cols, rows, cwd?, onEvent: Channel<PtyEvent>) -> id:u32` | 起 shell，返回会话 id |

### 3.3 ★新增：统一 Agent Harness

**3.3.1 归一化事件（`agent/event.rs`）**
```rust
use serde::Serialize;

#[derive(Serialize, Clone)]
// 修 P0(二轮)：`rename_all="camelCase"` 只改 variant 名（started/delta/...），
// 不改 struct variant 内的字段名。必须再加 `rename_all_fields="camelCase"`，
// 否则 `agent_session_id`/`cost_usd` 仍以 snake_case 序列化，与前端契约
// （agentSessionId/costUsd，见 §4.3）对不上 → resume 拿不到 id 而失效。
#[serde(tag = "kind", rename_all = "camelCase", rename_all_fields = "camelCase")]
pub enum AgentEvent {
    /// 会话开始；session_id 供多轮 resume（CC=session_id / CX=thread_id）
    Started { agent_session_id: Option<String> },
    /// 增量文本（打字机效果）
    Delta { text: String },
    /// 工具调用（UI 折叠展示）
    ToolUse { name: String, summary: Option<String> },
    /// 文件改动 → 触发 git diff 刷新
    FileChange { path: String },
    /// 完成
    Done { ok: bool, result: Option<String>, cost_usd: Option<f64> },
    /// 失败
    Failed { message: String },
}
```

**3.3.2 Harness trait + 三实现（`agent/harness.rs`）**

每个 harness 负责：① 构造 CLI 命令（含 §决策 D1 的直接写盘 flag）；② 把该 CLI 的原生 JSONL 行解析成 `AgentEvent`。

```rust
use crate::agent::event::AgentEvent;
use serde_json::Value;

pub enum AgentKind { Claude, Codex }   // CU(Cursor) 已砍（D2）

impl AgentKind {
    pub fn from_code(c: &str) -> Option<Self> {
        match c { "CC" => Some(Self::Claude), "CX" => Some(Self::Codex), _ => None }
    }
    pub fn bin(&self) -> &'static str {
        match self { Self::Claude=>"claude", Self::Codex=>"codex" }
    }

    /// 构造命令行参数（D1：均直接写盘）
    pub fn args(&self, prompt: &str, resume: Option<&str>) -> Vec<String> {
        match self {
            // CC: stream-json + 自动接受编辑 + 可 resume
            Self::Claude => {
                let mut a = vec![
                    "-p".into(), prompt.into(),
                    "--output-format".into(), "stream-json".into(),
                    "--verbose".into(), "--include-partial-messages".into(),
                    "--permission-mode".into(), "acceptEdits".into(),  // D1 直接写盘
                ];
                if let Some(sid) = resume { a.push("--resume".into()); a.push(sid.into()); }
                a
            }
            // CX: exec --json + workspace-write；resume 用子命令形式（见 spawn 注）
            Self::Codex => {
                let mut a = vec!["exec".into()];
                if let Some(tid) = resume { a.push("resume".into()); a.push(tid.into()); }
                a.extend([
                    prompt.into(),
                    "--json".into(),
                    "--sandbox".into(), "workspace-write".into(),  // D1 直接写盘
                    "--skip-git-repo-check".into(),                // P1：非 git 目录也能跑
                ]);
                a
            }
        }
    }

    /// 解析一行原生 JSONL → 归一化事件（None = 忽略该行）
    /// ⚠️ 字段名需对照 §1.5 钉定版本的 golden fixtures 校正；缺字段返回 None 不 panic。
    /// ⚠️ 修 P1-9(二轮)：`FileChange` 是**优化事件不是唯一来源**——CC 不稳定产出文件改动列表。
    ///    diff 刷新主路径靠"run 完成 + 窗口 focus + 会话激活 + commit/discard 后"（§4.4），
    ///    并在 agent 结束时用 baseline delta 合成 FileChange（§3.4.3），不依赖具体 CLI 字段。
    pub fn parse_line(&self, line: &str) -> Option<AgentEvent> {
        let v: Value = serde_json::from_str(line).ok()?;
        match self {
            Self::Claude => match v.get("type")?.as_str()? {
                "system" if v.get("subtype")?.as_str()? == "init" =>
                    Some(AgentEvent::Started { agent_session_id: v.get("session_id").and_then(|s| s.as_str()).map(String::from) }),
                "stream_event" => {
                    let d = v.get("event")?.get("delta")?;
                    if d.get("type")?.as_str()? == "text_delta" {
                        Some(AgentEvent::Delta { text: d.get("text")?.as_str()?.into() })
                    } else { None }
                }
                // 修 P1-9(二轮)：CC 的 tool_use 在 assistant 消息的 content 里，原来整段被忽略，
                // 导致"折叠 toolUse"对 CC 落空。这里取第一个 tool_use 块产出 ToolUse 事件。
                // （一条 assistant 可能含多个块；MVP 取首个工具调用，多工具二期遍历 content。）
                "assistant" => {
                    let content = v.get("message")?.get("content")?.as_array()?;
                    content.iter().find_map(|c| {
                        if c.get("type")?.as_str()? == "tool_use" {
                            Some(AgentEvent::ToolUse {
                                name: c.get("name").and_then(|s| s.as_str()).unwrap_or("tool").into(),
                                summary: c.get("input").map(|i| i.to_string()),
                            })
                        } else { None }
                    })
                }
                "result" => {
                    let ok = !v.get("is_error").and_then(|b| b.as_bool()).unwrap_or(false)
                             && v.get("subtype").and_then(|s| s.as_str()) == Some("success");
                    Some(AgentEvent::Done {
                        ok,
                        result: v.get("result").and_then(|s| s.as_str()).map(String::from),
                        cost_usd: v.get("total_cost_usd").and_then(|n| n.as_f64()),
                    })
                }
                _ => None,
            },
            Self::Codex => match v.get("type")?.as_str()? {
                "thread.started" =>
                    Some(AgentEvent::Started { agent_session_id: v.get("thread_id").and_then(|s| s.as_str()).map(String::from) }),
                "item.completed" => {
                    // file_change → FileChange；agent_message 文本 → Delta（CX 通常整段给）；
                    // command_execution / mcp_tool_call → ToolUse（修 P1-9：CX 工具调用也归一化）
                    let item = v.get("item")?;
                    match item.get("type")?.as_str()? {
                        "file_change" => Some(AgentEvent::FileChange {
                            path: item.get("path").and_then(|s| s.as_str()).unwrap_or("").into() }),
                        "agent_message" => Some(AgentEvent::Delta {
                            text: item.get("text").and_then(|s| s.as_str()).unwrap_or("").into() }),
                        "command_execution" => Some(AgentEvent::ToolUse {
                            name: "command".into(),
                            summary: item.get("command").and_then(|s| s.as_str()).map(String::from) }),
                        "mcp_tool_call" => Some(AgentEvent::ToolUse {
                            name: item.get("tool").and_then(|s| s.as_str()).unwrap_or("mcp").into(),
                            summary: None }),
                        _ => None,
                    }
                }
                "turn.completed" => Some(AgentEvent::Done { ok: true, result: None, cost_usd: None }),
                // 修 P1-9(二轮)：提取真实错误字段，不再硬编码 "turn failed" 丢失诊断。
                "turn.failed" => Some(AgentEvent::Failed {
                    message: v.get("error").and_then(|e| e.get("message")).and_then(|s| s.as_str())
                        .or_else(|| v.get("error").and_then(|s| s.as_str()))
                        .unwrap_or("turn failed").into() }),
                _ => None,
            },
        }
    }
}
```

**3.3.3 进程管理 + 流式推送（`agent/mod.rs`）**
```rust
use std::collections::HashMap;
use std::sync::atomic::{AtomicU32, AtomicUsize, Ordering};
use std::sync::{Arc, RwLock};
use std::time::Duration;
use tauri::ipc::Channel;
use tokio::io::{AsyncBufReadExt, BufReader};
use tokio::process::Command;
use tokio::sync::{OwnedSemaphorePermit, Semaphore};

use crate::agent::{event::AgentEvent, harness::AgentKind};

// D6：软上限可配。默认 4，运行时从设置（tauri-plugin-store）读。
const DEFAULT_MAX_CONCURRENT: usize = 4;
// 修 P0(二轮)：result-hang 兜底——总时长 + 静默上限。任一超时即 kill。
const WALL_CLOCK_TIMEOUT: Duration = Duration::from_secs(600);  // 单轮总时长上限
const IDLE_TIMEOUT: Duration       = Duration::from_secs(120);  // 无新事件静默上限

/// 每个在跑的 agent：持有取消句柄（kill 子进程）+ 占用的并发名额。
/// 名额随 RunningAgent 一起 drop 时自动归还（修 P1：超时/取消/完成三路径都归还）。
struct RunningAgent {
    killer: tokio::sync::Mutex<Option<tokio::process::Child>>,
    _permit: OwnedSemaphorePermit,   // drop = 释放并发名额
}

type Procs = Arc<RwLock<HashMap<u32, Arc<RunningAgent>>>>;

pub struct AgentState {
    procs: Procs,
    next_id: AtomicU32,
    sem: Arc<Semaphore>,             // 修 P1：并发用信号量，check+占额原子，无 TOCTOU
    max_configured: AtomicUsize,     // 仅用于「设置」展示当前上限值
}
impl Default for AgentState {
    fn default() -> Self {
        Self {
            procs: Arc::new(RwLock::new(HashMap::new())),
            next_id: AtomicU32::new(0),
            sem: Arc::new(Semaphore::new(DEFAULT_MAX_CONCURRENT)),
            max_configured: AtomicUsize::new(DEFAULT_MAX_CONCURRENT),
        }
    }
}
// 修 P0(二轮)：之前调用写成 `state.procs.clone_handle()`——procs 是 Arc<RwLock<..>>，
// 没有该方法。clone_handle 应挂在 AgentState 上，返回 procs 的 Arc 克隆。
impl AgentState { fn clone_handle(&self) -> Procs { self.procs.clone() } }

#[tauri::command]
pub async fn agent_spawn(
    state: tauri::State<'_, AgentState>,
    agent: String,          // "CC"|"CX"
    prompt: String,
    cwd: String,
    resume: Option<String>, // agent_session_id（CC=session_id / CX=thread_id）
    on_event: Channel<AgentEvent>,
) -> Result<u32, String> {
    // 修 P1(二轮)：D6 软上限用 Semaphore，check+占额原子完成——
    // 并发 spawn 不会都通过 len() 检查而突破上限。MVP=超限即拒绝（try_acquire），排队二期。
    let permit = state.sem.clone().try_acquire_owned()
        .map_err(|_| "并发上限：请等待其他会话完成或在设置中调高上限".to_string())?;

    let kind = AgentKind::from_code(&agent).ok_or("未知 agent")?;
    let mut child = Command::new(kind.bin())
        .args(kind.args(&prompt, resume.as_deref()))
        .current_dir(&cwd)
        .stdout(std::process::Stdio::piped())
        .stderr(std::process::Stdio::piped())   // 必须 drain，见下（修 P0：写满管道会 hang）
        .kill_on_drop(true)
        .spawn()
        .map_err(|e| format!("启动 {} 失败：{e}（是否已安装/登录/PATH 可见？见 §3.3.4）", kind.bin()))?;
        // permit 在此 ? 提前返回时随函数栈 drop，自动归还名额。

    let stdout = child.stdout.take().ok_or("no stdout")?;
    let stderr = child.stderr.take().ok_or("no stderr")?;
    let id = state.next_id.fetch_add(1, Ordering::Relaxed) + 1;
    // child + 名额都交给 RunningAgent；任务结束 remove 时一起 drop，名额归还。
    let running = Arc::new(RunningAgent {
        killer: tokio::sync::Mutex::new(Some(child)),
        _permit: permit,
    });
    state.procs.write().unwrap().insert(id, running.clone());

    // 修 P0：drain stderr（攒成失败诊断；否则管道写满整个 agent 卡死）
    let stderr_buf = Arc::new(tokio::sync::Mutex::new(String::new()));
    {
        let buf = stderr_buf.clone();
        tokio::spawn(async move {
            let mut lines = BufReader::new(stderr).lines();
            while let Ok(Some(l)) = lines.next_line().await {
                let mut g = buf.lock().await; g.push_str(&l); g.push('\n');
            }
        });
    }

    let ch = on_event.clone();
    let procs = state.clone_handle();
    tokio::spawn(async move {
        let mut lines = BufReader::new(stdout).lines();
        let mut saw_done = false;
        let mut timed_out = false;
        // 修 P0(二轮)：result-hang 真兜底。仅"EOF 后 wait"挡不住 stdout 不 EOF/进程不退的 hang。
        // 用 wall-clock + idle 双超时包住整个读取循环：超时 → kill → 标记失败。
        let wall = tokio::time::sleep(WALL_CLOCK_TIMEOUT);
        tokio::pin!(wall);
        loop {
            tokio::select! {
                // 逐行读 stdout → 解析 → 推前端
                line = lines.next_line() => match line {
                    Ok(Some(line)) => {
                        if let Some(ev) = kind.parse_line(&line) {
                            if matches!(ev, AgentEvent::Done{..} | AgentEvent::Failed{..}) { saw_done = true; }
                            let _ = ch.send(ev);
                        }
                    }
                    _ => break,   // EOF 或读错误：正常走兜底
                },
                // 静默超时：IDLE_TIMEOUT 内无任何新行
                _ = tokio::time::sleep(IDLE_TIMEOUT) => { timed_out = true; break; }
                // 总时长超时
                _ = &mut wall => { timed_out = true; break; }
            }
        }

        let mut guard = running.killer.lock().await;
        if timed_out {
            // 超时：主动 kill；EOF/正常退出：仍 wait 取真实退出码
            if let Some(c) = guard.as_mut() { let _ = c.start_kill(); let _ = c.wait().await; }
            let _ = ch.send(AgentEvent::Failed {
                message: "agent 超时未完成，已终止（可能 hang，见 §3.3.3/§8）".into() });
        } else {
            let status = match guard.as_mut() { Some(c) => c.wait().await.ok(), None => None };
            let ok = status.map(|s| s.success()).unwrap_or(false);
            // 兜底：CC 偶发 result 缺失（issue #1920/#8126/#25629）——没见过 Done 才补发
            if !saw_done {
                let msg = stderr_buf.lock().await.clone();
                if ok { let _ = ch.send(AgentEvent::Done { ok: true, result: None, cost_usd: None }); }
                else  { let _ = ch.send(AgentEvent::Failed { message:
                            if msg.is_empty() { "agent 异常退出".into() } else { msg } }); }
            }
        }
        drop(guard);
        // 从 procs 移除 → RunningAgent drop → 名额（_permit）归还。超时/正常/取消同一出口。
        procs.write().unwrap().remove(&id);
    });
    Ok(id)
}

#[tauri::command]
pub async fn agent_cancel(state: tauri::State<'_, AgentState>, id: u32) -> Result<(), String> {
    // 取句柄并 kill；读取任务会在循环 break 后走同一清理出口 remove（名额随之归还）。
    let running = state.procs.read().unwrap().get(&id).cloned();
    if let Some(r) = running {
        if let Some(c) = r.killer.lock().await.as_mut() { let _ = c.start_kill(); }
    }
    Ok(())
}

// 多轮对话主路径 = 用 resume 重新 spawn（CC --resume / CX exec resume），
// 不走 agent stdin。故 §3.1 的 invoke_handler 不注册 agent_write，前端也不暴露该契约。
// 若未来要做"同一进程内多轮"，需另行实现：spawn 时 .stdin(piped())、持有 ChildStdin、
// 提供写入/换行/关闭与对应取消语义——非 MVP。
```

```rust
use serde::Serialize;
```

**3.4.1 读：status / diff / ahead-behind（`git/mod.rs`，用 git2）**

> 修 P1：要覆盖 **staged(HEAD→index) + unstaged(index→workdir) + untracked**，并给出**每文件** `+x −y`，不留 `0/0` 占位。做法：对 workdir 做一次 `diff_tree_to_workdir_with_index`（含 HEAD tree 比对 + untracked），用 per-file patch 累计行数。

```rust
use std::collections::HashMap;
use git2::{Repository, DiffOptions};
use serde::Serialize;

#[derive(Serialize)]
pub struct FileChange { pub path: String, pub status: String, pub added: usize, pub removed: usize }
#[derive(Serialize)]
pub struct StatusResult { pub branch: String, pub files: Vec<FileChange>, pub summary: String }

#[tauri::command]
pub fn git_status(repo_path: String) -> Result<StatusResult, String> {
    let repo = Repository::discover(&repo_path).map_err(|e| e.to_string())?;
    let branch = repo.head().ok()
        .and_then(|h| h.shorthand().map(String::from))
        .unwrap_or_else(|| "HEAD".into());

    // HEAD tree → workdir（经 index），含 untracked，覆盖 staged+unstaged 全部改动
    let head_tree = repo.head().ok()
        .and_then(|h| h.peel_to_tree().ok());
    let mut opts = DiffOptions::new();
    opts.include_untracked(true).recurse_untracked_dirs(true);
    let diff = repo.diff_tree_to_workdir_with_index(head_tree.as_ref(), Some(&mut opts))
        .map_err(|e| e.to_string())?;

    // 修 P2-15(二轮)：先用 file callback 登记每个 delta，再用 line callback 累计增删。
    // 只靠 line callback 会漏掉零行变更——空文件新增、纯 mode change、rename 没有 +/- 行，
    // 这些文件就不会出现在列表里。foreach 的 file_cb 对每个 delta 必触发一次。
    let mut per_file: HashMap<String, (usize, usize, char)> = HashMap::new();
    let mark_of = |d: &git2::DiffDelta| match d.status() {
        git2::Delta::Added | git2::Delta::Untracked => 'A',
        git2::Delta::Deleted => 'D',
        git2::Delta::Renamed => 'R',
        _ => 'M',
    };
    diff.foreach(
        &mut |delta, _progress| {   // file_cb：先登记，保证零行变更也在列表里
            let path = delta.new_file().path().or_else(|| delta.old_file().path())
                .map(|p| p.to_string_lossy().into_owned()).unwrap_or_default();
            per_file.entry(path).or_insert((0, 0, mark_of(&delta)));
            true
        },
        None, None,
        Some(&mut |delta, _hunk, line| {   // line_cb：累计增删
            let path = delta.new_file().path().or_else(|| delta.old_file().path())
                .map(|p| p.to_string_lossy().into_owned()).unwrap_or_default();
            let e = per_file.entry(path).or_insert((0, 0, mark_of(&delta)));
            match line.origin() { '+' => e.0 += 1, '-' => e.1 += 1, _ => {} }
            true
        }),
    ).map_err(|e| e.to_string())?;

    let mut files: Vec<FileChange> = per_file.into_iter()
        .map(|(path, (added, removed, mark))| FileChange { path, status: mark.to_string(), added, removed })
        .collect();
    files.sort_by(|a, b| a.path.cmp(&b.path));
    let (ta, tr): (usize, usize) = files.iter().fold((0, 0), |(a, r), f| (a + f.added, r + f.removed));
    let summary = format!("{} 文件 · +{} −{}", files.len(), ta, tr);
    Ok(StatusResult { branch, files, summary })
}

// 修 P1-13(二轮)：git_diff 不返回裸字符串，返回结构化结果 + 硬边界，
// 避免二进制/超大/minified 文件拖垮 300px mini diff。
const DIFF_MAX_BYTES: usize = 256 * 1024;   // 单文件 patch 上限
const DIFF_MAX_LINES: usize = 2000;         // 单文件行数上限（mini diff 视图再自行截到 ~300）

#[derive(Serialize)]
#[serde(tag = "kind", rename_all = "camelCase")]
pub enum FileDiff {
    Text { path: String, patch: String, truncated: bool, total_lines: usize },
    Binary { path: String },
    TooLarge { path: String, bytes: usize },
    MetadataOnly { path: String, change: String },  // rename / mode / emptyFile
}

#[tauri::command]
pub fn git_diff(repo_path: String, file: String) -> Result<FileDiff, String> {
    let repo = Repository::discover(&repo_path).map_err(|e| e.to_string())?;
    let head_tree = repo.head().ok().and_then(|h| h.peel_to_tree().ok());
    let mut opts = DiffOptions::new();
    opts.include_untracked(true).pathspec(&file);
    let diff = repo.diff_tree_to_workdir_with_index(head_tree.as_ref(), Some(&mut opts))
        .map_err(|e| e.to_string())?;

    // 先看 delta：二进制 / 纯 metadata 直接短路，不走 patch 文本
    if let Some(delta) = diff.deltas().next() {
        if delta.flags().is_binary() {
            return Ok(FileDiff::Binary { path: file });
        }
        // 零字节新增 / 纯 mode change / rename 可能无 line 内容（与 P2-15 同源）
        match delta.status() {
            git2::Delta::Renamed => return Ok(FileDiff::MetadataOnly { path: file, change: "rename".into() }),
            git2::Delta::Typechange => return Ok(FileDiff::MetadataOnly { path: file, change: "mode".into() }),
            _ => {}
        }
    }

    let mut out = String::new();
    let mut lines = 0usize;
    let mut truncated = false;
    diff.print(git2::DiffFormat::Patch, |_d, _h, l| {
        if out.len() >= DIFF_MAX_BYTES || lines >= DIFF_MAX_LINES { truncated = true; return true; }
        let origin = l.origin();
        if matches!(origin, '+'|'-'|' ') { out.push(origin); lines += 1; }
        out.push_str(std::str::from_utf8(l.content()).unwrap_or(""));
        true
    }).map_err(|e| e.to_string())?;

    if truncated && out.len() >= DIFF_MAX_BYTES {
        return Ok(FileDiff::TooLarge { path: file, bytes: out.len() });
    }
    Ok(FileDiff::Text { path: file, patch: out, truncated, total_lines: lines })
}

// 修 P1-10(二轮)：ahead-behind 不返回裸 (usize,usize)/Err——无 upstream、detached、unborn
// 都结构化降级，绝不连累 diff 文件列表。前端按 state 显示而非报错。
#[derive(Serialize)]
#[serde(tag = "state", rename_all = "camelCase")]
pub enum RemoteState {
    Ok { upstream: String, ahead: usize, behind: usize },
    NoUpstream { branch: String },
    Detached { oid: String },
    Unborn,                       // 新仓库还没 commit
    Unknown { message: String },
}

#[tauri::command]
pub fn git_ahead_behind(repo_path: String) -> Result<RemoteState, String> {
    let repo = Repository::discover(&repo_path).map_err(|e| e.to_string())?;
    let head = match repo.head() {
        Ok(h) => h,
        // unborn：HEAD 指向尚不存在的分支（git init 后未提交）
        Err(ref e) if e.code() == git2::ErrorCode::UnbornBranch => return Ok(RemoteState::Unborn),
        Err(e) => return Ok(RemoteState::Unknown { message: e.to_string() }),
    };
    if repo.head_detached().unwrap_or(false) {
        return Ok(RemoteState::Detached {
            oid: head.target().map(|o| o.to_string()).unwrap_or_default() });
    }
    let local = match head.target() { Some(o) => o, None => return Ok(RemoteState::Unborn) };
    let branch = head.shorthand().unwrap_or("HEAD").to_string();
    let upstream = match repo.find_branch(&branch, git2::BranchType::Local).and_then(|b| b.upstream()) {
        Ok(u) => u,
        Err(_) => return Ok(RemoteState::NoUpstream { branch }),  // 无 upstream → 降级，不报错
    };
    let up_name = upstream.name().ok().flatten().unwrap_or("").to_string();
    let up_oid = match upstream.get().target() { Some(o) => o, None => return Ok(RemoteState::NoUpstream { branch }) };
    match repo.graph_ahead_behind(local, up_oid) {
        Ok((ahead, behind)) => Ok(RemoteState::Ok { upstream: up_name, ahead, behind }),
        Err(e) => Ok(RemoteState::Unknown { message: e.to_string() }),
    }
}
```

**3.4.2 写：commit + push 全走系统 `git` CLI（D4 + P2-7）（`git/commit.rs`）**
```rust
use std::time::Duration;
use tokio::process::Command;

const GIT_TIMEOUT: Duration = Duration::from_secs(60);

/// 统一跑 git 子命令：禁交互（不挂起等密码）+ 超时**真 kill** + 捕获 stderr 回 UI。
/// 修 P1-7(二轮)：不再用 `timeout(Command::output())`——那只丢弃 future，底层 git/ssh 进程
/// 可能继续存活留下残留。改为 spawn + kill_on_drop + select，超时分支显式 kill+wait。
async fn run_git(repo_path: &str, args: &[&str]) -> Result<String, String> {
    let mut child = Command::new("git")
        .current_dir(repo_path)
        .args(args)
        .env("GIT_TERMINAL_PROMPT", "0")    // P2-7：要凭证时立即失败而非挂起
        // 如需进一步禁 ssh 交互：.env("GIT_SSH_COMMAND", "ssh -oBatchMode=yes")
        .stdout(std::process::Stdio::piped())
        .stderr(std::process::Stdio::piped())
        .kill_on_drop(true)                 // 兜底：handle 被 drop 时也会 kill
        .spawn()
        .map_err(|e| format!("无法执行 git：{e}（是否已安装 git / PATH 可见？见 §3.3.4）"))?;

    let out = tokio::select! {
        res = child.wait_with_output() =>
            res.map_err(|e| format!("git 执行失败：{e}"))?,
        _ = tokio::time::sleep(GIT_TIMEOUT) => {
            // 超时：显式 kill + 回收，杜绝残留进程
            let _ = child.start_kill();
            let _ = child.wait().await;
            return Err("git 超时（可能在等待凭证/网络），已终止。请先在终端完成登录或凭证配置。".into());
        }
    };
    if out.status.success() {
        Ok(String::from_utf8_lossy(&out.stdout).into_owned())
    } else {
        // stderr 原样回显（如 "Authentication failed" / "could not read Username"）
        Err(String::from_utf8_lossy(&out.stderr).into_owned())
    }
}

#[tauri::command]
pub async fn git_commit(repo_path: String, message: String) -> Result<String, String> {
    // 暂存全部改动后提交。`git commit` 自动继承 user.name/email、pre-commit hooks、GPG 签名。
    run_git(&repo_path, &["add", "-A"]).await?;
    run_git(&repo_path, &["commit", "-m", &message]).await
    // 失败场景由 run_git 回显：无改动可提交、pre-commit hook 拒绝等。
}

#[tauri::command]
pub async fn git_push(repo_path: String) -> Result<(), String> {
    run_git(&repo_path, &["push"]).await.map(|_| ())
    // 凭证复用系统 credential helper / ssh-agent；需登录时 stderr 提示回 UI。
}
```

**3.4.3 ★新增：本轮改动模型 `AgentRunBaseline`（修 P0/P1，二轮——D1 直接写盘的安全底座）**

D1 让 agent 直接写盘，核心风险随之从"apply patch"转移到 **区分 agent 本轮改动与用户原有 dirty work**。这是数据模型问题，不是 UI 文案问题。`agent_spawn` 前记录 baseline，结束后算出本轮 delta：
```rust
use std::collections::HashMap;
use std::path::PathBuf;
use git2::Oid;

/// spawn 前对仓库拍快照。仅指纹，不拷贝内容。
pub struct AgentRunBaseline {
    pub repo_root: PathBuf,
    pub head_oid: Option<Oid>,            // unborn 仓库为 None
    pub index_tree_oid: Option<Oid>,
    pub tracked: HashMap<PathBuf, FileFingerprint>,    // 路径 → (mtime,len,mode/oid)
    pub untracked: HashMap<PathBuf, FileFingerprint>,
    pub captured_at: i64,
}

/// 结束后用"baseline vs 当前"分类，驱动「丢弃本次 Agent 改动」与 commit 范围。
pub struct AgentRunDelta {
    pub agent_only: Vec<PathBuf>,    // 启动前干净、本轮 agent 新增 → 可安全展示/丢弃
    pub pre_existing: Vec<PathBuf>,  // 启动前已有 → 永不自动丢弃
    pub conflicted: Vec<PathBuf>,    // 同一路径启动前已改、agent 又改 → 拒绝自动丢弃，提示人工
    pub external: Vec<PathBuf>,      // 运行期间检测到的预期外修改 → 降级人工
}
```
- 「丢弃改动」只回滚 `agent_only`；`conflicted`/`pre_existing` 禁止一键操作（UI 见 §4.4）。
- **commit 范围（修 P1，二轮）**：当前 `git_commit` 走 `add -A` + `commit`，会把 `pre_existing` 一起提交。MVP 至少 UI 文案写明"提交当前工作区全部改动"；更优雅是默认提交 `agent_only`、`pre_existing` 单列让用户勾选"也包含"、`conflicted` 存在时禁止一键提交（见 §10 P1 与新 review）。
- **MVP 取舍**：完整四分类是目标态；MVP 至少落 `agent_only` vs `pre_existing` 两分类 + `conflicted` 拒绝路径，`external` 二期。

### 3.7 ★新增：统一基础设施（合并自《实现优化-review》— 子进程/PATH 抽象）

前面 agent / git / preflight 三类子进程，各自都要处理 stdout-drain、stderr-drain、timeout、cancel、kill_on_drop、超时显式 kill+wait、错误回显。若每个 command 手写一份，必然行为漂移（agent 超时能 kill、git 不能；agent drain stderr、preflight 不 drain；cancel 释放名额、timeout 忘了释放）。把它们抽成**两个底座**，所有 CLI 调用共用。

**3.7.1 `ProcessRunner`（`modules/process/`）—— 子进程生命周期统一出口**
```text
src-tauri/src/modules/process/
├── mod.rs
├── runner.rs        # run_capture（跑完拿 stdout/stderr/status）/ run_streaming_lines（流式 + drain + cancel）
├── managed_child.rs # ManagedChild：kill / wait / kill_on_drop guard
└── error.rs         # ProcessError → 统一 UI 错误
```
```rust
// git / preflight：一把梭拿结果
pub async fn run_capture(spec: CommandSpec, timeout: Duration)
    -> Result<CommandOutput, ProcessError>;

// agent：流式读 stdout 行 + 同时 drain stderr + 支持外部 cancel
pub fn run_streaming_lines(spec: CommandSpec, policy: TimeoutPolicy, on_line: impl FnMut(String))
    -> ManagedProcessHandle;

pub struct TimeoutPolicy {                 // agent 至少要三者
    pub wall_clock: Option<Duration>,      // 总时长上限
    pub idle: Option<Duration>,            // 无新事件静默上限
    pub cancel: CancelToken,               // 用户手动取消
}
```
统一终止出口（§3.3.3 的 agent、§3.4.2 的 git 都收敛到这里）：
```text
timeout / cancel / EOF / exit
        │
        ▼
kill if needed → wait → emit terminal event → cleanup registry → persist final state
```
> 收益：行为天然一致，且 fake-CLI 测试只需针对 runner 写一次。§3.3.3 的 `agent_spawn` 与 §3.4.2 的 `run_git` 视为该底座的两个调用点（文中保留展开实现以示语义，落地时统一走 runner）。

**3.7.2 `CommandResolver`（修 P0-4，二轮）—— macOS 打包后找得到 CLI**
`cargo tauri dev`（继承终端 PATH）能找到 `claude/codex/git`，**不代表 Finder 双击 `.app` 能找到**——macOS GUI app 通常不继承用户 shell PATH。这不是"CLI 未装"，是 app bundle 环境问题。**不要**把 PATH 修复散落在 `agent_preflight` / `run_git` 里，启动时统一做：
```rust
// setup() 中尽早调用一次，结果存进全局状态
pub fn resolve_bin(name: &str) -> ResolvedCommand;   // claude/codex/git

pub struct ResolvedCommand { pub path: Option<PathBuf>, pub source: ResolveSource }
pub enum ResolveSource { UserOverride, LoginShellPath, SystemPath, NotFound }
```
解析优先级：① 设置里的**用户绝对路径覆盖** → ② 从 login shell 取 PATH（`fix-path-env` crate，或 `$SHELL -ilc 'echo $PATH'`）→ ③ 合并常见路径 `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin`。设置页展示解析结果：
```text
Claude Code: /opt/homebrew/bin/claude   ✓ 已登录
Codex:       未找到                       设置路径…
Git:         /usr/bin/git                ✓
```
> ⚠️ 验收必须包含**打包后双击 `.app` 启动**，不能只测 dev 模式（§7 / M6）。`agent_preflight`、`run_git`、`agent_spawn` 启动子进程时一律走 `resolve_bin` 的结果，不再裸 `Command::new("claude")`。

**3.7.3 推荐实现顺序（合并自新 review）**
先底座、再协议、再 harness、再 git、最后 UI polish——比"先拼 UI 再到处补后端边界"稳：
1. **基础设施**：`ProcessRunner` / `CommandResolver` / `AgentRunBaseline` 三个底座（§3.7.1/3.7.2/3.4.3）。
2. **协议**：`AgentEvent` wire protocol + serde rename（§3.3.1）+ TS 类型（§4.3）+ wire snapshot 测试。
3. **agent harness**：CC/CX parser 只做事件归一化，不碰 UI 状态（§3.3.2）。
4. **Git 面板**：diff 与 remote status 分离、支持降级（§3.4.1）。
5. **UI polish**：ShellSessionView / AgentTranscriptView 分流、delta batch、diff limits（§4.4）。

### 4.2 会话编排 store（`state/sessions.ts`）
```ts
export type AgentCode = 'CC'|'CX';                 // CU 已移除（D2）
export type SessionStatus = 'running'|'fresh'|'failed'|'done';
export type SessionKind = 'shell'|'agent';

// 内联块：少量结构（toolUse/fileChange/轮次分隔）。正文 delta 不进数组，见 reply。
export type AgentBlock =
  | { type:'toolUse'; name:string; summary?:string }
  | { type:'fileChange'; path:string }
  | { type:'turn' };

export type Session = {
  id: string;
  kind: SessionKind;
  agent?: AgentCode;
  agentSessionId?: string;          // CC=session_id / CX=thread_id（多轮 resume）
  procId?: number;                  // 后端 agent_spawn / pty_open 返回的 id
  title: string;
  dir: string; branch: string;
  status: SessionStatus;
  // ⚠️ 不要把每条 delta 存成数组（长会话 O(n²) 复制）。正文累积进字符串：
  reply: string;                    // 打字机正文（仅追加）
  blocks: AgentBlock[];             // 少量块结构
  result?: string; error?: string; updatedAt: number;  // 终态（D5：必持久化）
  changes?: { files: FileChange[]; summary: string; commit?: string };
};

// 分组：按 dir 归组 → 设计稿侧边栏 ~/orbit ~/web ~/infra
export const groupByDir = (s: Session[]) =>
  s.reduce<Record<string, Session[]>>((acc, x) => ((acc[x.dir] ??= []).push(x), acc), {});

// 状态机（被 agent 事件驱动）。注意：delta 只 += 字符串，不复制数组。
export function applyEvent(s: Session, ev: AgentEvent): Session {
  const now = Date.now();
  switch (ev.kind) {
    case 'started':   return { ...s, status:'running', updatedAt:now,
                               agentSessionId: ev.agentSessionId ?? s.agentSessionId };
    case 'delta':     return { ...s, reply: s.reply + ev.text, updatedAt:now };
    // ⚠️ 修 P2-14(二轮)：真成本不在字符串拼接（引擎多为 rope，追加接近 O(1)），
    //    而在**每个 token 都触发 setState → re-render + diff**。故 delta 不要逐条进 applyEvent，
    //    要在事件入口用 mutable chunk buffer 攒着，requestAnimationFrame（或 50–100ms）
    //    合并成一个 chunk 再 append（实现见 §4.4 onDelta）。applyEvent 只处理"块"级事件。
    case 'toolUse':   return { ...s, blocks:[...s.blocks, {type:'toolUse', name:ev.name, summary:ev.summary}], updatedAt:now };
    case 'fileChange':return { ...s, blocks:[...s.blocks, {type:'fileChange', path:ev.path}], updatedAt:now }; // + 去抖刷 git（§4.4）
    case 'done':      return { ...s, status: ev.ok ? 'fresh':'failed', result: ev.result, updatedAt:now };
    case 'failed':    return { ...s, status:'failed', error: ev.message, updatedAt:now };
  }
}
// fresh → done：用户切走 或 N 秒后降级（UI 层定时）。
// 终态持久化（修 P2-16，二轮）：不要只在 done/failed 落 store——`fresh→done` 是 UI 定时降级，
//   若不落盘，重启后会继续错显"刚完成"。对策：store 存**稳定事实**而非易变的 UI 态：
//     runState: 'running'|'completed'|'failed'|'cancelled'|'timeout'
//     startedAt / completedAt / error? / resultSummary? / unread?
//   UI 的 `fresh` 由事实**派生**：fresh = runState==='completed'
//                                && Date.now()-completedAt < FRESH_WINDOW && unread。
//   这样重启 / 通知未读数 / fresh→done 降级三者天然一致（详见 §10 P2 + 新 review「Store 存事实」）。
```

### 4.3 Agent 前端桥（`modules/agent/agent-bridge.ts`）

```ts
import { invoke, Channel } from "@tauri-apps/api/core";
export type AgentEvent =
```

### 4.4 终端区 + 渲染模型 + diff 刷新

**渲染模型分层（修 P0-5，二轮——MVP 必须先拍板）**
设计稿要"AI 回复内联在终端流里"，但 **xterm.js 的 buffer/scrollback/selection/resize 不是普通 DOM，React 块插不进 xterm 输出流**。所以**不存在**"先用 React 写块再塞进 xterm"这条路。MVP 把渲染边界写死成两条独立路径，由 `kind` 分流：
```text
SessionViewport
├── ShellSessionView      # kind=shell：内部只有 xterm（复用 terax useTerminalSession）
└── AgentTranscriptView   # kind=agent：纯 React transcript（reply 打字机 + blocks），不伪装成 xterm
```
- agent 会话是 React transcript，复用设计稿的字体/背景/提示符风格，但**不承诺 xterm buffer 内联**。
- 原命名 `InlineAgentBlock.tsx` 易误导，归入 `AgentTranscriptView`；若保留该文件名，注明"仅在 AgentTranscriptView 内联，不是 xterm buffer 内联"。
- **目标形态（二期/风险项）**：外层 `BlockTerminalView` 虚拟列表承载 `ShellCommandBlock`（已完成命令的只读快照）+ `AgentBlock`（React）+ `ActiveXtermBlock`（当前交互 xterm）。这最贴设计稿，但要做命令边界（OSC 133 切块）、跨 block 滚动/选择/复制/搜索/resize/历史重放，**代价高、不进 MVP**（详见调研稿 §1.3 / 深挖稿 §1.3）。

**真实终端（shell）**：复用 terax `useTerminalSession`（xterm + webgl，cursorStyle `bar` = 设计稿"竖条"）；OSC 7 → 状态栏 cwd、OSC 133 → 命令边界。

**AgentTranscriptView 的打字机（修 P2-14，二轮）**：delta 不逐条 setState，事件入口用 chunk buffer + rAF 合并：
```ts
const pending = useRef('');
const raf = useRef<number | null>(null);
function onDelta(text: string) {           // Channel 收到 delta 时调用
  pending.current += text;
  if (raf.current == null) {
    raf.current = requestAnimationFrame(() => {
      const chunk = pending.current; pending.current = ''; raf.current = null;
      appendReplyChunk(sessionId, chunk);  // 一帧只 setState 一次
    });
  }
}
// 极密集时可换 50–100ms 节流。视觉仍是打字机，但 CPU/re-render 稳定。
```

**「丢弃改动」绑定 baseline（修 P1-8，二轮）**：D1 直接写盘后，最大风险是误伤用户启动 agent 前已有的 dirty work。**不要**直接 `git checkout -- <file>` / `git stash`——那会连用户原有改动一起回滚。正确做法绑定 §3.4.3 的 `AgentRunBaseline`：
- 按钮语义改为「**丢弃本次 Agent 改动**」（不是"丢弃这个文件"）。
- 只回滚 `agentOnlyChanges`（启动前干净、本轮 agent 新增）。
- 命中 `preExistingChanges`（启动前已有）：**永不自动丢弃**。
- 命中 `conflictedChanges`（同一路径启动前已改、agent 又改）：按钮禁用 + 提示"该文件在 Agent 启动前已有改动，无法安全区分本轮改动，请手动处理"。

**diff 刷新触发点（修 P1-9，二轮——`fileChange` 是优化事件不是唯一来源）**：CC 不稳定产出文件改动事件，**不能只靠 fileChange**。以下任一都刷新 `DiffPanel`：① **agent run 完成**（必刷，用 §3.4.3 baseline delta 合成 `FileChange[]`，不依赖 CLI 字段）；② **会话切换/激活**；③ **窗口 focus**（捕捉手动改文件 / `git checkout` / 外部编辑器）；④ **commit / 丢弃完成后**；⑤ agent `fileChange` 事件若有（`debounce 300ms`，作为提前刷新的加速）。更完整方案二期接 `notify` 文件 watcher。刷新 = `git_status` + 首文件 `git_diff`。

## 5. 关键数据流（端到端）

```
新建 Agent:
  UI 选目录+agent+prompt → preflightAgent()(存在/登录) → 记录 AgentRunBaseline(§3.4.3)
   → spawnAgent(agent,prompt,cwd,resume?)
   → [Rust] resolve_bin(§3.7.2) → spawn(Semaphore 占额,原子) → 逐行读 stdout（+drain stderr）
   → parse_line → AgentEvent(camelCase) → Channel.send → [UI] onDelta(chunk+rAF)/applyEvent
   → reply/blocks 更新 AgentTranscriptView；done→fresh(绿勾)/failed/timeout→通知（落 store 事实字段）
   → 终止出口统一(EOF/exit/wall-clock/idle timeout/cancel)：kill if needed→wait→补终态→remove(名额随 _permit drop 归还)
   → run 完成用 baseline delta 合成 FileChange[] → 刷 git

真实终端（复用）:
  term.onData → pty_write ;  pty Data 事件(base64) → term.write ;  resize → pty_resize

Git 面板:
  会话激活 / 窗口 focus / run 完成 / commit·丢弃后 → git_status + git_ahead_behind(结构化降级) → 渲染文件卡/remote 状态
  点文件 → git_diff(file)→FileDiff(text/binary/tooLarge/metadataOnly) → mini diff 染色
  提交 → git_commit(msg)[CLI: add -A + commit] ; 提交并推送 → git_commit + git_push[CLI]
  丢弃 → 仅回滚 baseline 的 agent_only；conflicted/pre_existing 拒绝自动处理
```

## 8. 风险与对策速查

| 风险 | 对策 |
|---|---|
| CLI 字段/版本漂移 | `parse_line` 容错（缺字段返回 None 不 panic）；钉版本 + golden fixtures 回归（§1.5） |

| agent stderr 写满管道 hang | drain stderr 任务（§3.3.3） |
| agent 完成不释放并发名额 | 名额（`OwnedSemaphorePermit`）随 RunningAgent drop 归还；三终止路径同一出口 remove（§3.3.3） |

| 大仓 git 慢 / **单文件超大·二进制 diff** | 刷新去抖 300ms + 后台；`git_diff` 返回结构化 FileDiff + 行/字节上限 + binary 标记（§3.4.1，修 P1-13） |

| 受保护路径写入 | 优先阻断/失败提示，仅实际高风险改动持久通知（深挖稿 §1.3，修 P1） |
