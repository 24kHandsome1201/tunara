## 二、确认缺失的功能

以下功能经代码全量搜索确认**不存在**，按优先级排序。

### 2.1 可点击 URL 检测

**对标：** Ghostty / iTerm2 / WezTerm / Warp / Kitty / Alacritty — 全部支持。

**现状：** `@xterm/addon-web-links` 未安装（不在 `package.json`），无任何 URL 点击处理。终端输出中的 http/https/file 链接无法点击。

**影响：** 终端基本功能，没有会被认为是半成品。开发者每天都会在终端输出中看到 URL（错误堆栈里的文档链接、CI 输出的构建地址、git remote URL）。

**实现方案：**

```bash
pnpm add @xterm/addon-web-links
```

`terminal-instance.ts` 或 `TerminalView.tsx` 初始化时加载：

```typescript
import { WebLinksAddon } from "@xterm/addon-web-links";
import { open } from "@tauri-apps/plugin-opener";

term.loadAddon(new WebLinksAddon((_, uri) => {
  open(uri);
}));
```

`@tauri-apps/plugin-opener` 已在 `package.json` 中，直接使用。

**工作量：** ~20 行。

---

### 2.2 Bell / 完成通知

**对标：** iTerm2（Dock bounce + 系统通知）、Ghostty（Dock badge）、WezTerm（可配置通知）。

**现状：** `BEL = "\x07"` 仅在 `terminal-input-buffer.ts` 中作为 OSC 分隔符使用，无用户可感知的 bell 处理。已有 `session.unread` 标记（侧栏内可见），但窗口不在前台时用户完全不知道 agent 跑完了。

**影响：** Agent 跑完了但 Conduit 在后台 → 核心场景断裂。这是 Conduit 作为 AI agent 终端的关键缺失——用户切到浏览器查资料，agent 完成了却无提醒。

**实现方案：**

1. **xterm `onBell` 监听：** 终端收到 `\x07` 时触发
2. **Dock bounce：** 窗口非聚焦时调用 `getCurrentWindow().requestUserAttention(UserAttentionType.Informational)`（Tauri 已内置，无需额外插件）
3. **可选系统通知：** 通过 `tauri-plugin-notification` 推送（"Claude Code 已完成 · ~/code/project"）
4. **Settings 开关：** `ui.ts` 加 `bellNotification: boolean`，默认开启

额外加分项：agent 从 `running` → `done`/`failed` 时也触发通知（不依赖 bell 字符），覆盖所有 agent CLI。

**工作量：** ~80 行（监听 + Dock + 可选通知 + Settings 开关）。

---

### 2.3 Scrollback 可配置

**对标：** 所有前沿终端都支持配置，iTerm2 默认无限、Ghostty 默认 10000、WezTerm 默认 3500。

**现状：** `terminal-instance.ts:33` 硬编码 `scrollback: 3_000`，`ui.ts` 无对应 state，`Settings.tsx` 无入口。

**影响：** 3000 行对短命令够用，但 agent 长时间运行（Claude Code 一次任务可能输出上万行）时会丢失早期输出，无法回滚查看。

**实现方案：**

1. `ui.ts` 新增 `scrollback: number`（默认 3000），进 `PERSIST_KEYS`
2. `Settings.tsx` 外观区加滑块（范围 1000–10000，步进 1000）
3. `createTerminalInstance` 读取 store 值
4. `useTerminalRuntimeSync` 在 scrollback 变化时更新已有实例（`term.options.scrollback = value`）

**工作量：** ~30 行。

---

### 2.4 字体连字（Ligatures）

**对标：** Ghostty / WezTerm / iTerm2 / Kitty 均支持。

**现状：** `@xterm/addon-ligatures` 未安装，JetBrains Mono 自带的连字字形（`=>` `!==` `->` `>=`）不会渲染。

**影响：** 视觉品味层面。JetBrains Mono 的连字是该字体的标志性特征，不渲染等于浪费了字体选择。对"好看"定位的终端来说是减分项。

**实现方案：**

xterm.js v6 的连字支持有两条路径：

1. **`@xterm/addon-ligatures`（社区维护）：** 安装即用，但依赖 `font-ligatures` npm 包（~500KB），且在 WebGL 渲染模式下可能有兼容问题
2. **等待 xterm.js 原生支持：** xterm.js 团队在讨论原生连字，但截至 2026 年中仍未合并

建议先尝试方案 1，如果 WebGL addon 冲突则降级为 canvas 渲染时启用连字。

**工作量：** ~20 行代码，但需要测试 WebGL 兼容性。

**风险：** `addon-ligatures` 可能与 `addon-webgl` 冲突（已知 issue），需要实测。如果冲突，要么放弃连字，要么在启用连字时降级为 canvas 渲染（性能略降但可接受）。

### 2.5 选中即复制 / 智能选词

**对标：** iTerm2（双击选词可配置分隔符 + 选中自动复制）、Ghostty（智能选词）、WezTerm（可配置 `word_delimiters`）。

**现状：** 无自定义 `wordSeparator`、无 `onSelectionChange` 监听。xterm.js 默认行为：双击在 `/` `.` `-` 处断开，无法选中完整文件路径。

**影响：** 中等。开发者频繁双击选路径/URL 然后复制，默认分隔符导致选不中完整内容。

**实现方案：**

```typescript
// terminal-instance.ts — createTerminalInstance 内
wordSeparator: ' ()[]{}\'",;',  // 去掉 / . - : 让双击能选整段路径
```

可选增强：Settings 加「选中自动复制」开关：

```typescript
term.onSelectionChange(() => {
  const sel = term.getSelection();
  if (sel) navigator.clipboard.writeText(sel);
});
```

**工作量：** 基础 ~5 行（wordSeparator），含自动复制 ~25 行。

## 三、与前沿终端的对比矩阵

### 3.1 vs 带侧栏终端（直接竞品）

| 功能 | Conduit | Warp | cmux |
|------|---------|------|------|
| 侧栏会话管理 | ✅ 按目录分组 + 折叠 + 右键 + 重命名 | ✅ | ✅ |
| Agent 检测 | ✅ 11 种自动识别 | ✅ 自研 | ✅ |
| Git Diff 审查 | ✅ 内置 DiffPanel | ❌ | ❌ |
| 文件浏览 | ✅ FileExplorer + Preview | ❌ | ❌ |
| Split Pane | ✅ 2 pane | ✅ 无限 | ✅ 无限 |
| Command Palette | ✅ 含权重排序 | ✅ | ❌ |
| URL 点击 | ❌ | ✅ | ✅ |
| 完成通知 | ❌ | ✅ | ✅ |
| 跨平台 | macOS only | macOS/Linux/Windows | macOS |
| 体积 | ~30MB | ~150MB | ~50MB |

**判断：** Conduit 在审查维度（DiffPanel + FileExplorer）上比两个竞品都强，但 URL 点击和完成通知的缺失让它在基础终端功能上弱于两者。

**判断：** 纯终端能力上 Conduit 弱于三者，但这些终端都没有侧栏会话管理、Agent 检测、DiffPanel。Conduit 的差异化不在终端引擎层面。需要补的是那些「没有会觉得是 bug」的基础功能（URL、通知、scrollback 配置）。

## 四、优先级排序与建议

### 第一梯队：必须做（没有会被视为缺陷）

| 序号 | 功能 | 工作量 | 理由 |
|------|------|--------|------|
| 1 | 可点击 URL | ~20 行 | 所有终端都有，缺了是明显 bug 级体验 |
| 2 | Bell / 完成通知 | ~80 行 | Agent 跑完了用户不知道 = 核心场景断裂 |
| 3 | Scrollback 可配置 | ~30 行 | 硬编码 3000 对 agent 长输出不够，且 Settings 里缺这一项显得不完整 |

**合计 ~130 行，半天工作量。**

### 第二梯队：应该做（提升品味）

| 序号 | 功能 | 工作量 | 理由 |
|------|------|--------|------|
| 4 | 智能选词（wordSeparator） | ~5 行 | 一行配置，ROI 极高 |
| 5 | 字体连字 | ~20 行 + 测试 | "好看"定位的终端不渲染连字是矛盾的 |

### 第三梯队：可以做（锦上添花）

| 序号 | 功能 | 工作量 | 理由 |
|------|------|--------|------|
| 6 | 选中自动复制（可选） | ~20 行 | iTerm2 用户的肌肉记忆 |
| 7 | 终端字体粗细配置 | ~15 行 | 完善 Settings |
| 8 | 图片协议 | 大 | 边缘需求，xterm.js 第三方 addon 不成熟 |
| 9 | 快捷键可配置 | 大 | 成熟终端标志，但当前快捷键数量少，不急 |

### 明确不做

| 功能 | 理由 |
|------|------|
| 递归 tile layout | 产品定位已决 |
| 自研渲染引擎 | xterm.js + WebGL 够用 |
| 插件系统 | 太早期 |
| SSH / 远程连接 | 不做运维 |
| 内置 AI 聊天 | 不做 AI 产品 |
| 700+ 主题库 | 7 套精选覆盖主流审美，不追数量 |

## 五、结论

Conduit 作为「AI Agent 侧栏终端」的差异化功能（DiffPanel + FileExplorer + Agent 检测 + 状态条 + 会话管理）已经比 Warp 和 cmux 做得更深。产品完成度约 85 分。

剩余 15 分集中在「作为日常终端的基本功」上——URL 点击、Bell 通知、Scrollback 配置、智能选词。这四项合计 ~135 行代码、不超过一天的工作量，补上后 Conduit 就是一个可以完全替代 iTerm2 日常使用的、比 Warp 轻比 cmux 好看的现代终端。
