# 设置面板修复与增强

> 审查范围：Settings.tsx、tokens.css（light/dark）、terminalTheme.ts、useTheme.ts、ui.ts state
> 审查标准：WCAG AA（文本 4.5:1、UI 元素 3:1）

---

## 一、逻辑 Bug

### 1.1 强调色派生色不跟随（P0）

**现状**：用户在设置面板选择非 Terracotta 的强调色（如 Indigo #4f6ef0）后，`--c-accent` 被正确更新，但以下 CSS 变量仍然固定为 Terracotta 的派生色：

| 变量 | Light 固定值 | Dark 固定值 |
|------|-------------|-------------|
| `--c-accent-bg-light` | `#fbeadf`（暖棕） | `#3a2a20` |
| `--c-accent-bg-soft` | `#fbf6f2` | `#2e231c` |
| `--c-accent-border` | `#f1d6c6` | `#5a3e2e` |

**表现**：选 Indigo 蓝色强调色 → 侧栏选中项 hover 背景仍是暖棕色。全局不一致。

**同源问题**：终端 `selectionBackground` 在 LIGHT_THEME 硬编码 `#c2683c44`、DARK_THEME 硬编码 `#e0907066`，不跟强调色走。

**修复方向**：`useTheme.ts` 中根据 `accent` 色值动态计算派生色并写入 CSS 变量。终端 selection 色也从 accent 派生。

### 1.2 Sand 强调色浅色模式下不可用（P0）

**现状**：Sand `#e2c08d` 用作选中环时在白色背景上对比度仅 **1.73:1**（要求 ≥3:1）。

**表现**：用户选 Sand 后，AccentRing 选中态的 border 和 box-shadow 在浅色模式下几乎不可见，选中反馈消失。

**修复方向**：将 Sand 色值调深至约 `#c4a060` 或类似值，保证在白底上 ≥3:1。

### 1.3「默认」终端配色预览失真（P1）

**现状**：Settings.tsx 中「默认」选项的缩略图硬编码 `bg: "#18181b", fg: "#e4e4e7"`（深色），但浅色 app 主题下「默认」实际渲染为白底终端（LIGHT_THEME `background: "#ffffff"`）。

**表现**：浅色主题用户看到 5 个全是深色底的预览，误以为选「默认」也是深色终端。

**修复方向**：「默认」选项的缩略图颜色根据当前 app 主题动态切换（浅色时显示白底预览，深色时显示深底预览）。

## 二、对比度问题

### 2.1 全局文字色（P0）

| 场景 | 前景 | 背景 | 实测对比度 | 要求 |
|------|------|------|-----------|------|
| Light: `text-5` 在 `bg-white` | `#a1a1aa` | `#ffffff` | 2.56:1 | ≥3:1 |
| Dark: `text-5` 在 `bg-3` | `#52525b` | `#2a2a2e` | 1.85:1 | ≥3:1 |
| Dark: `text-4` 在 `bg-3` | `#71717a` | `#2a2a2e` | 2.96:1 | ≥3:1 |

`text-5` 大量用于状态文字、"加载中"、"已截断"、文件路径等实际可读内容，不是纯装饰。

**修复方向**：
- Light `text-5`：`#a1a1aa` → `#8e8e96`（约 3.2:1）
- Dark `text-5`：`#52525b` → `#65656e`（约 3.1:1）
- Dark `text-4`：`#71717a` → `#7a7a84`（约 3.2:1）

### 2.2 Accent 作为文字（P1）

| 场景 | 前景 | 背景 | 实测 | 要求 |
|------|------|------|------|------|
| Light: accent 在白底 | `#c2683c` | `#ffffff` | 3.93:1 | ≥4.5:1（文本） |
| Light: accent 在 accent-bg-light | `#c2683c` | `#fbeadf` | 3.36:1 | ≥4.5:1 |

accent 用作 Agent badge 文字、强调链接等。修复 1.1 后派生色会变化，需一并验证。

### 2.3 Diff 配色（P1）

| Light: diff-del-text 在 diff-del-bg | `#c0414e` | `#fcebec` | 4.44:1 |

**修复方向**：
- diff-add-text：`#1f8a5b` → `#1a7a4f`
- diff-del-text：`#c0414e` → `#b03a46`

### 2.4 终端主题内部（P1）

| 主题 | 问题色 | 背景 | 实测 |
|------|--------|------|------|
| Solarized | brightGreen `#586e75` | `#002b36` | 2.79:1 |
| Solarized | brightYellow `#657b83` | `#002b36` | 3.37:1 |
| Solarized | black `#073642` | `#002b36` | 1.15:1 |
| One Dark | brightRed `#be5046` | `#282c34` | 2.95:1 |

Solarized 的 bright 色是原版设计问题（base01 被映射为 brightGreen）。One Dark 的 brightRed 低于标准。

**修复方向**：这些是社区配色方案的固有问题，可以不改（保持原味），但如果改：Solarized brightGreen 用 `#859900`（原版 green），One Dark brightRed 用 `#e06c75`（原版 red）。

## 三、设计问题

### 3.1 终端配色与 App 主题分离

**现状**：
- 4 个非默认终端配色（Catppuccin、Tokyo Night、One Dark、Solarized）全是纯深色
- `getTerminalTheme()` 对非 default 直接忽略 `appTheme` 参数
- 浅色 app 主题下选它们 → 终端区深色、UI 区浅色
- 面板中无任何说明文字

**评估**：终端和 UI 独立配色是专业终端的标配设计（iTerm2、Warp、Hyper 都这样做），分离本身合理。但缺少两个东西：
1. 用户引导 — 一句说明文字："仅影响终端区域"
2. 浅色系终端配色 — 没有 Solarized Light、Catppuccin Latte 等选项

**建议**：
- 终端配色区域 label 下加一行辅助文字
- 长期可考虑加 1-2 个浅色终端配色，短期不阻塞

### 3.2 设置面板缺少的 Tab/区域

当前只有「外观」和「CLI」两个 tab。CLI tab 是纯信息展示（已安装 CLI 列表），不可调。
