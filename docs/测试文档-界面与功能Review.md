# Tunara 界面与功能 Review 测试文档

> 日期：2026-06-19  
> 范围：当前三栏主界面、标题栏、侧边栏、终端会话、审查面板、设置页、Agent CLI 识别与窗口行为。


- 测试仓库中保留一个干净 Git 工作区，再准备一个有未提交改动的工作区，用于审查面板与关闭/重启场景。

| SS-01 | 新建终端按钮 | 点击侧栏“+ 新建终端” | 新会话使用当前会话目录；立即成为激活 Tab；侧栏计数增加 |

## 4. 终端与 Agent CLI 自动识别

> 当前行为是“在 shell 会话里输入特定 CLI 命令后，把该终端会话标记成对应 Agent”。这不是新建独立 Agent transcript 的流程，因此需要明确验证它只做识别与展示，不破坏终端输入。

| ID | 场景 | 操作 | 预期 |
|---|---|---|---|
| AG-01 | Claude Code 识别 | 在终端输入 `claude` 后回车 | 当前会话标题更新为 `Claude Code`；侧栏/Tab 显示 CC 对应角标；命令仍正常发送给 PTY |
| AG-02 | Codex 识别 | 在新终端输入 `codex` 后回车 | 当前会话标题更新为 `Codex`；显示 CX 角标 |
| AG-03 | Amp 识别 | 在新终端输入 `amp` 或 `ampcode` 后回车 | 当前会话标题更新为 `Amp`；显示 AM 角标和 Amp 配色 |
| AG-04 | 带参数识别 | 输入 `claude --help`、`codex exec --help`、`amp --version` | 只取首个 token 判断，仍能识别对应 Agent |
| AG-05 | 退格修正 | 输入 `claudex`，退格改成 `claude` 后回车 | 最终按修正后的命令识别，不因中间输入误判 |
| AG-06 | 非目标命令 | 输入 `echo claude` 或 `pnpm claude` | 不标记为 Agent，因为首个 token 不是目标 CLI |
| AG-07 | 单会话只触发一次 | 同一终端先输入 `claude`，再输入 `codex` | 首次识别后不反复改写会话标题/agent 类型 |
| AG-08 | 粘贴命令 | 粘贴 `amp --help` 后回车 | 识别 Amp；若终端粘贴发送包含控制序列，至少不能破坏输入或抛前端异常 |

### 4.1 需要特别关注的风险

- 识别逻辑只看前端输入流，不能证明 CLI 真正启动成功；失败时应由终端原始输出呈现。
- 当前识别发生在 shell 会话上，不会自动创建 `kind = agent` 的 transcript 会话；如果产品预期是“启动 Agent 后切成对话视图”，这是需求差异，需要单独实现。

## 6. 审查面板与 Git 功能

| ID | 场景 | 操作 | 预期 |
|---|---|---|---|
| RV-01 | 打开审查面板 | 点击标题栏审查按钮 | 右栏出现 DiffPanel；关闭后主区自适应 |
| RV-02 | 干净工作区 | 在无改动仓库打开审查面板 | 显示“无改动”或等价空状态，不报错 |
| RV-03 | 有文本改动 | 修改一个文本文件后刷新/切回会话 | 文件列表与增删行统计正确；patch 可读 |
| RV-04 | 非 Git 目录 | 在非 Git 目录新建终端并打开审查面板 | 面板降级提示，不影响终端使用 |
| RV-05 | 无 upstream / detached | 在无 upstream 或 detached HEAD 仓库打开 | 本地 diff 仍可展示；远端状态单独提示失败原因 |
| RV-06 | 大文件/二进制 | 添加二进制文件或大 diff | 不应卡死 UI；若未实现截断，应记录为当前缺口 |

## 9. 当前 review 结论与待确认项

### 已覆盖的变更点

- 标题栏设置入口从侧栏迁移到右上角。
- 侧栏移除“新建 Agent”分段，只保留“新建终端”。
- 终端输入 `claude` / `codex` / `amp` / `ampcode` 后自动标记 Agent 类型。
- Agent 类型新增 Amp，并补齐角标配色与设置页展示。
- Tauri 主窗口启用原生 decorations，并补充 minimize / toggle-maximize 权限。

### 待产品确认

1. **Agent 自动识别只基于用户输入命令，不基于进程实际状态。** 这适合轻量标记，但不等价于 Agent 会话生命周期管理。
2. **一次识别后不再切换 Agent 类型。** 如果用户在同一 shell 里先后运行多个 Agent CLI，需要决定是否允许再次识别。
3. **当前产品边界是“带智能侧栏的终端”。** 若未来恢复 prompt 式新建 Agent，需要重新评估是否会把产品拉回 Agent 管理平台。
